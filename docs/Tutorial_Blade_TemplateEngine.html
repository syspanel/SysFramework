Tutorial Completo do Blade Templating Engine

Um guia abrangente para entender e usar o Blade com exemplos detalhados.

Introdução

Blade é um motor de template poderoso e fácil de usar, incluído no Laravel. Ele oferece uma sintaxe expressiva e recursos avançados para facilitar a criação de views dinâmicas e reutilizáveis.

@extends

A diretiva @extends permite que você defina um layout base que outras views podem "herdar".

@extends('layouts.master')

Exemplo: Um layout base layouts/master.blade.php pode ser estendido em outras views:

@extends('layouts.master')

@section e @endsection

A diretiva @section define uma seção de conteúdo que pode ser inserida no layout pai.

@section('content')
    <h1>Título da Página</h1>
@endsection

Exemplo: Defina conteúdo específico em uma seção:

@extends('layouts.master')

@section('content')
    <h1>Título da Página</h1>
    <p>Este é o conteúdo da página.</p>
@endsection

@yield

A diretiva @yield é usada dentro de layouts para indicar onde o conteúdo de uma seção deve ser inserido.

@yield('content')

Exemplo: No layout base, você pode ter algo assim:

<body>
    @yield('content')
</body>

@if, @elseif, @else e @endif

Essas diretivas permitem a criação de condições em suas views.

@if($user)
    <p>Bem-vindo, {{ $user->name }}</p>
@elseif($guest)
    <p>Bem-vindo, Convidado</p>
@else
    <p>Por favor, faça login</p>
@endif

Exemplo: Exiba diferentes conteúdos com base em uma condição:

@if($user)
    <p>Bem-vindo, {{ $user->name }}</p>
@elseif($guest)
    <p>Bem-vindo, Convidado</p>
@else
    <p>Por favor, faça login</p>
@endif

@isset e @endisset

A diretiva @isset verifica se uma variável foi definida e não é null.

@isset($record)
    <p>O recorde é {{ $record }}</p>
@endisset

Exemplo: Exiba conteúdo apenas se uma variável estiver definida:

@isset($record)
    <p>O recorde é {{ $record }}</p>
@endisset

@empty e @endempty

A diretiva @empty verifica se uma variável está vazia.

@empty($records)
    <p>Não há registros disponíveis</p>
@endempty

Exemplo: Exiba uma mensagem se não houver registros disponíveis:

@empty($records)
    <p>Não há registros disponíveis</p>
@endempty

@for e @endfor

A diretiva @for permite criar loops baseados em uma condição.

@for($i = 0; $i < 10; $i++)
    <p>Item {{ $i }}</p>
@endfor

Exemplo: Um loop simples que gera uma lista de itens:

@for($i = 0; $i < 10; $i++)
    <p>Item {{ $i }}</p>
@endfor

@foreach e @endforeach

A diretiva @foreach é usada para iterar sobre arrays ou coleções.

@foreach($items as $item)
    <p>Item: {{ $item }}</p>
@endforeach

Exemplo: Itere sobre uma lista de itens:

@foreach($items as $item)
    <p>Item: {{ $item }}</p>
@endforeach

@forelse e @endforelse

A diretiva @forelse é usada para iterar sobre uma coleção, mas também pode fornecer uma alternativa se a coleção estiver vazia.

@forelse($users as $user)
    <p>{{ $user->name }}</p>
@empty
    <p>Nenhum usuário encontrado</p>
@endforelse

Exemplo: Exiba uma mensagem alternativa se não houver usuários:

@forelse($users as $user)
    <p>{{ $user->name }}</p>
@empty
    <p>Nenhum usuário encontrado</p>
@endforelse

@while e @endwhile

A diretiva @while cria um loop baseado em uma expressão booleana.

@while($condition)
    <p>Este é um loop while</p>
@endwhile

Exemplo: Um loop simples enquanto a condição for verdadeira:

@while($condition)
    <p>Este é um loop while</p>
@endwhile

@include

A diretiva @include permite incluir uma view dentro de outra.

@include('partials.header')

Exemplo: Incluindo um arquivo de cabeçalho:

@include('partials.header')

@php e @endphp

A diretiva @php permite que você insira código PHP diretamente na view.

@php
    $currentTime = date('H:i:s');
@endphp

Exemplo: Executando código PHP dentro de uma view:

@php
    $currentTime = date('H:i:s');
@endphp

<p>Hora atual: {{ $currentTime }}</p>

@csrf

A diretiva @csrf é usada para inserir um token CSRF em formulários.

@csrf

Exemplo: Incluindo um token CSRF em um formulário:

<form method="POST" action="/login">
    @csrf
    <input type="text" name="email">
    <input type="password" name="password">
    <button type="submit">Login</button>
</form>

@method

A diretiva @method é usada para especificar métodos HTTP diferentes de POST em formulários.

@method('PUT')

Exemplo: Criando um formulário com método PUT:

<form method="POST" action="/update">
    @method('PUT')
    @csrf
    <input type="text" name="email">
    <button type="submit">Atualizar</button>
</form>

@error

A diretiva @error é usada para exibir mensagens de erro relacionadas a campos de formulário.

@error('email')
    <div class="alert alert-danger">{{ $message }}</div>
@enderror

Exemplo: Exibindo uma mensagem de erro para o campo de e-mail:

<input type="text" name="email">
@error('email')
    <div class="alert alert-danger">{{ $message }}</div>
@enderror

@auth e @guest

Essas diretivas são usadas para verificar a autenticação do usuário. @auth verifica se o usuário está autenticado, e @guest verifica se o usuário não está autenticado.

@auth
    <p>Bem-vindo, {{ auth()->user()->name }}</p>
@endauth

@guest
    <p>Bem-vindo, visitante!</p>
@endguest

Exemplo: Exibindo conteúdo diferente para usuários autenticados e convidados:

@auth
    <p>Bem-vindo, {{ auth()->user()->name }}</p>
@endauth

@guest
    <p>Bem-vindo, visitante!</p>
@endguest

@switch, @case, @break e @default

Essas diretivas funcionam como um switch-case em PHP.

@switch($i)
    @case(1)
        <p>Primeiro caso</p>
        @break

    @case(2)
        <p>Segundo caso</p>
        @break

    @default
        <p>Caso padrão</p>
@endswitch

Exemplo: Criando um switch-case para diferentes valores:

@switch($i)
    @case(1)
        <p>Primeiro caso</p>
        @break

    @case(2)
        <p>Segundo caso</p>
        @break

    @default
        <p>Caso padrão</p>
@endswitch

@inject

A diretiva @inject permite injetar uma instância de um serviço diretamente em uma view.

@inject('service', 'App\Services\MyService')

Exemplo: Injetando um serviço em uma view:

@inject('service', 'App\Services\MyService')

<p>Resultado do serviço: {{ $service->perform() }}</p>

@json

A diretiva @json converte um array ou objeto em JSON e o exibe na view.

@json($array)

Exemplo: Exibindo um array como JSON:

@json(['name' => 'Laravel', 'version' => 8])
