Explicações Detalhadas

app/

Este diretório contém a lógica principal da aplicação.

  • Controllers/: Armazena os controladores, que gerenciam as requisições HTTP e retornam respostas apropriadas.
  • Models/: Contém os modelos que representam a estrutura de dados e interagem com o banco de dados.
  • Middlewares/: Contém middlewares que processam as requisições antes de chegar ao controlador ou após a resposta.
  • Console/: Armazena comandos que podem ser executados a partir da linha de comando.
  • Services/: Implementa a lógica de negócios, separando-a dos controladores.
  • UseCases/: Define casos de uso específicos da aplicação, encapsulando processos de negócios complexos.
  • Events/: Define eventos que podem ser disparados dentro da aplicação.
  • Listeners/: Contém ouvintes que respondem a eventos específicos, permitindo uma arquitetura orientada a eventos.

config/

Armazena os arquivos de configuração da aplicação.

  • bootstrap.php: Script de inicialização da aplicação, carregando configurações essenciais.
  • paths.php: Define os caminhos utilizados na aplicação, como diretórios de recursos.
  • settings.php: Contém configurações gerais da aplicação, como modo de depuração.
  • database.php: Configurações de conexão e opções do banco de dados.

core/

Contém o núcleo do framework, onde estão as classes e utilitários principais.

  • SysTemplateEngine.php: Classe responsável pelo mecanismo de templates.
  • SysRouter.php: Classe que gerencia as rotas da aplicação.
  • SysORM.php: Sistema de mapeamento objeto-relacional para interagir com o banco de dados.
  • SysModel.php: Classe base para os modelos, oferecendo funcionalidades padrão.
  • SysController.php: Classe base para os controladores, oferecendo métodos utilitários comuns.
  • helpers/: Funções auxiliares que podem ser usadas em várias partes da aplicação.
  • utils/: Utilitários e ferramentas que auxiliam no funcionamento do framework.

public/

Diretório acessível publicamente. Contém arquivos que podem ser servidos diretamente pelo servidor web.

  • index.php: Ponto de entrada principal da aplicação. Todas as requisições passam por este arquivo.
  • assets/: Armazena recursos estáticos como CSS, JavaScript, e imagens.

resources/

Contém arquivos de recursos que não são acessíveis diretamente, mas usados na aplicação.

  • views/: Armazena os templates e views que serão renderizados pela aplicação.

logs/

Armazena arquivos de log da aplicação, úteis para depuração e monitoramento.

  • app.log: Arquivo de log principal da aplicação, onde são registradas mensagens de erro, avisos, etc.

caches/

Diretório para armazenar arquivos de cache, melhorando o desempenho da aplicação.

  • template.php: Arquivo de cache de templates, permitindo renderização mais rápida.

database/

Contém arquivos relacionados ao banco de dados.

  • migrations/: Armazena scripts de migração do banco de dados, gerenciando alterações no esquema.
  • seeders/: Scripts para popular o banco de dados com dados iniciais ou de teste.

tests/

Diretório dedicado para testes automatizados da aplicação.

  • Unit/: Contém testes unitários que verificam funcionalidades específicas de classes ou métodos.
  • Integration/: Armazena testes de integração que verificam como diferentes partes do sistema funcionam juntas.
  • Feature/: Testes de funcionalidade completos que simulam o comportamento do usuário.

Outros Arquivos

  • .env: Arquivo de variáveis de ambiente, contendo configurações que variam entre diferentes ambientes (desenvolvimento, produção).
  • cli.php: Script de entrada para comandos CLI (Command Line Interface) específicos da aplicação.
  • composer.json: Arquivo de configuração do Composer, gerenciando as dependências da aplicação.
